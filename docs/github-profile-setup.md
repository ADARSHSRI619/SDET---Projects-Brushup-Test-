# GitHub Profile Setup (So It Actually Reflects Publicly)

If your GitHub profile looks empty, this is usually due to **missing public repos**, **no profile README**, or **nothing pinned**.

## 1) Create your profile README repo (required)
GitHub only shows a profile README if you create a repository with the **exact same name as your username**.

Example:
- Username: `johnqa`
- Repository name: `johnqa`
- Add `README.md` in that repo root.

Use `profile/README-template.md` from this project as starter content.

## 2) Split this monorepo into 5 visible repositories
Create these public repos and push content into each:
1. `qa-api-automation`
2. `qa-ui-automation-playwright`
3. `qa-contract-testing-pact`
4. `qa-performance-k6`
5. `qa-engineering-handbook`

## 3) Pin repositories on your profile
On your GitHub profile:
- Click **Customize your pins**
- Pin the 5 repos above
- Keep most active/recent repo first

## 4) Add evidence artifacts
Recruiters trust proof over text. Add:
- CI status badge
- test reports
- sample failure screenshots/videos
- short architecture diagrams

## 5) Keep activity visible
- Push 2–3 commits weekly
- Open issues + PRs (even in your own repos)
- Tag releases (`v0.1.0`, `v0.2.0`)

## 6) Common reasons profile still looks empty
- Repo is private
- No pinned repos
- No profile README repo named exactly as username
- No commits on default branch
